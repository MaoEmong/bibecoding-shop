# 요구사항 정의 (현재 구현 기준)

작성일: 2026-02-06

## 범위
- 간단한 쇼핑몰 웹사이트를 포트폴리오 목적으로 구현합니다.
- 본 문서는 현재 코드에 구현된 기능과 차기 확장 기능을 구분해 기록합니다.

## 사용자 유형
- 판매자: 상품을 등록/관리하는 사용자
- 구매자: 상품을 탐색/구매하는 사용자

## 현재 구현된 요구사항

### 인증/계정
- 회원가입
- 로그인
- 로그아웃
- 회원가입 시 역할(판매자/구매자) 선택
- 로그인/회원가입 성공 시 세션 기반 인증 사용

### 판매자 기능
- 상품 등록
- 상품 수정(가격/재고/설명)
- 상품 삭제
- 본인 상품 목록 조회
- 판매자 주문 목록 조회
- 주문 상태 변경

### 구매자 기능
- 상품 목록 조회
- 상품 상세 보기
- 장바구니 담기/삭제
- 장바구니 수량 변경
- 주문 생성
- 주문 내역 조회
- 주문 상세 조회
- 주문 완료 페이지 조회

## 현재 미구현(확장 예정)
- 상품 검색/필터
- 구매 취소
- 배송 상태 상세 추적
- 찜하기(위시리스트)
- 문의 작성/관리
- 리뷰 작성/관리

## 비고
- API 인증은 토큰이 아니라 세션(`JSESSIONID`) 기반입니다.
- 상세 API 스펙은 `.docs/api.md`를 따릅니다.
