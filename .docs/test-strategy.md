# 테스트 전략

## 목표
- MVP 기능이 정상 동작하는지 확인합니다.
- 인증/권한/주문 흐름의 핵심 경로를 우선 검증합니다.

## 테스트 범위
- 단위 테스트: 서비스/도메인 로직
- 통합 테스트: 컨트롤러 + 서비스 + DB
- 인수 테스트(간단): 주요 사용자 흐름 시나리오

## 핵심 시나리오(MVP)
- 회원가입/로그인/로그아웃
- 판매자 상품 등록
- 상품 목록/상세 조회
- 장바구니 담기/수량 변경/삭제
- 주문 생성 및 내역 조회
- 판매자 주문 상태 변경

## 권한/보안
- SELLER/BUYER 권한 분리 확인
- 로그인 필수 API 접근 제어 확인

## 에러 케이스
- 존재하지 않는 상품/장바구니/주문 조회
- 재고 부족 시 주문 실패
- 권한 없는 API 호출

## 도구/구현
- Spring Boot 테스트(JUnit)
- @SpringBootTest 또는 MockMvc 활용
