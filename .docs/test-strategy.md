# 테스트 전략 (현재 코드 기준)

## 목표
- 현재 구현된 MVP 핵심 흐름이 정상 동작하는지 검증합니다.
- 권한/예외 케이스 테스트를 우선 보강 대상으로 둡니다.

## 현재 테스트 현황
- `MyprojectApplicationTests#contextLoads`
  - 스프링 컨텍스트 로딩 확인
- `ApiIntegrationTests#mvpFlowServices`
  - 서비스 계층 통합 흐름 확인(회원/상품/장바구니/주문/주문상태)

## 미흡한 영역(보강 필요)
- 인터셉터 권한 테스트(BUYER/SELLER/미인증)
- 예외 케이스 테스트(재고 부족, 리소스 없음, 권한 불일치)
- 페이지 컨트롤러(redirect/flash/model) 테스트
- API 응답 필드 정합성 테스트

## 권장 우선순위
1. 권한 우회 가능성 높은 주문 상태 변경 케이스
2. 장바구니/주문 실패 케이스
3. 페이지 라우팅 및 세션 기반 동작 검증
4. 회귀 테스트 셋 정리
